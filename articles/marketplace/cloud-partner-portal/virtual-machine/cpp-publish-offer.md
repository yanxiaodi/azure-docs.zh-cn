# <a name="publish-a-virtual-machine-offer"></a>发布虚拟机产品/服务

 在门户中定义产品/服务并创建关联的技术资产后，最后一个步骤是提交产品/服务进行发布。 下图描述了进行“推出”的发布过程的主要步骤：

![虚拟机产品/服务的发布步骤](./media/publishvm_013.png)

下表介绍了这些步骤并提供了完成它们所需的最大估计时间：
<!-- we need to tell them that if an offer seems stuck in a step, to know that they should file a support ticket (link to support ticket doc) -->


|  **发布步骤**           | **时间**    | **说明**                                                            |
|  -------------------           | --------    | ---------------                                                            |
| 验证先决条件         | 15 分钟   | 验证套餐信息和套餐设置。                        |
| 体验版验证（可选） | 2 小时 | 如果你已选择启用体验版，则 Microsoft 会验证体验版配置、其部署以及通过所选区域的复制。 |
| 认证                  | 3 天 | Azure 认证团队对产品/服务进行分析。 此步骤将针对病毒、恶意软件、安全合规性和安全问题进行扫描。 如果发现了问题，则提供反馈。 |
| 设置                   | 4 天   | 在市场生产系统中复制 VM 产品/服务。               |
| 打包以及潜在顾客开发注册 | \< 1 小时  | 打包产品/服务的技术资产供客户使用，配置并安装潜在顾客系统。 |
|  发布者签收             |  -        | 最终发布者在产品/服务推出前进行复查和确认。 可以在所选订阅（在套餐信息步骤中）中部署套餐，以验证它是否符合所有要求。  |
| 设置                   | 4 天 | 在市场生产系统和区域中复制最终完成的 VM 产品/服务。 | 
| 实时                           | 4 天 | 发布 VM 产品/服务，将其复制到所需的区域，并使其可供公众使用。 |
|  |  |

留出最多 16 天来完成此过程。  完成这些发布步骤后，你的 VM 产品/服务将在 [Microsoft Azure 市场](https://azuremarketplace.microsoft.com/marketplace/)中列出。
